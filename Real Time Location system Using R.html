Real Time Location System¶

By Lauren Darr, Heidi Nguyen, Ghizlane Zerouali¶

Introduction¶

Indoor positioning systems (IPS) can determine the location of an object or human by using a mobile device such as a cellphone or laptop to record signals to a known WiFi access point (routers). These hardware devices produce signals that provide location data. This location data is captured by other hardware devices that have softwares which use triangulation or trilateration algorithms to intepret this location data to identify the location of the object.

This case study was designed by Deborah Nolan and Duncan Temple Lang for "Data Science in R: A Case Studies Approach to Computational Reasoning and Problem Solving". The goal of this exercise is to utilize sensor data recorded inside a building by researchers at the University of Manneheim to develop a statistical model that can determine where an individual is in a building based on signal strength. In this case, the training data set was developed by taking millions of measurements of signal strength from known distances to routers in the building. This training data was explored and used in a K-Nearest Neighbors (KNN) approach in order to predict where a new individual may be based on signal data.

KNN is a non-parametric classification algorithm that is useful when not much is known about a data set. KNN is a lazy learner. This means that the training data set is not used to create a generalized model. Instead, each testing data point is compared to the training data set and then assigned the class it is closest to. It is assumed in KNN that a test point most likely belongs to the class of training points it is closest to.

Data Information and Methods¶

The wireless data was collected by a hand-held device that measured signal strength from a variety of distances and 8 different orientations to 6 access points. These 6 access points are used for the k-nearest neighbors model. Repetitive measurements of signal strength were taken at each known (calibration) location. The floor plan of the floor is detailed in Nolan's and Lang's book and is assumed to be accurate.

In their analysis, Nolan and Temple Lang use an IPS using signal strength data and a mean-based k-nearest neighbors model. The model is used to predict the device location using unseen signal strength data.

In this study, we validate Nolan and Temple Langs decision to exclude an access point from their indoor positioning system. We also implement an ## alternative ## prediction method k-nearest neighbors model and compare it to the mean-based k-nearest neighbors model that Nolan and Temple Lang developed.

The raw data is split into a training data set and a test data set. The training data set contains unique lines detailing the (x,y) position and orientation of a mobile device. Each position and orientation combination has 110 signal strength readings.POSIX time of the reading and the physical MAC address for each access point are also captured.

# function verification
readData = 
  function(filename = '../Downloads/offline.final.trace.txt', 
           subMacs = c("00:0f:a3:39:e1:c0", "00:0f:a3:39:dd:cd", "00:14:bf:b1:97:8a",
                       "00:14:bf:3b:c7:c6", "00:14:bf:b1:97:90", "00:14:bf:b1:97:8d",
                       "00:14:bf:b1:97:81"))
  {
    txt = readLines(filename)
    lines = txt[ substr(txt, 1, 1) != "#" ]
    tmp = lapply(lines, processLine)
    offline = as.data.frame(do.call("rbind", tmp), 
                            stringsAsFactors= FALSE) 
    
    names(offline) = c("time", "scanMac", 
                       "posX", "posY", "posZ", "orientation", 
                       "mac", "signal", "channel", "type")
    
     # keep only signals from access points
    offline = offline[ offline$type == "3", ]
    
    # drop scanMac, posZ, channel, and type - no info in them
    dropVars = c("scanMac", "posZ", "channel", "type")
    offline = offline[ , !( names(offline) %in% dropVars ) ]
    
    # drop more unwanted access points
    offline = offline[ offline$mac %in% subMacs, ]
    
    # convert numeric values
    numVars = c("time", "posX", "posY", "orientation", "signal")
    offline[ numVars ] = lapply(offline[ numVars ], as.numeric)

    # convert time to POSIX
    offline$rawTime = offline$time
    offline$time = offline$time/1000
    class(offline$time) = c("POSIXt", "POSIXct")
    
    # round orientations to nearest 45
    offline$angle = roundOrientation(offline$orientation)
      
    return(offline)
  }

There are three angles are considered when building the training data used for the k-nearest neighbors model. Thus, for every new prediction of location, a training set is created using three orientations. For our weighted k-nearest neighbors model, we also use three orientations.

# select angles and reshape the training set
# signals is the training dataset to filter and reshape
selectTrain = function(angleNewObs, signals = NULL, m = 1){
  # m is the number of angles to keep between 1 and 5
  refs = seq(0, by = 45, length  = 8)
  nearestAngle = roundOrientation(angleNewObs)
  
  if (m %% 2 == 1) 
    angles = seq(-45 * (m - 1) /2, 45 * (m - 1) /2, length = m)
  else {
    m = m + 1
    angles = seq(-45 * (m - 1) /2, 45 * (m - 1) /2, length = m)
    if (sign(angleNewObs - nearestAngle) > -1) 
      angles = angles[ -1 ]
    else 
      angles = angles[ -m ]
  }
  angles = angles + nearestAngle
  angles[angles < 0] = angles[ angles < 0 ] + 360
  angles[angles > 360] = angles[ angles > 360 ] - 360
  angles = sort(angles) 
  
  offlineSubset = signals[ signals$angle %in% angles, ]
  reshapeSS(offlineSubset, varSignal = "avgSignal")
}

findNN = function(newSignal, trainSubset) {
  diffs = apply(trainSubset[ , 4:9], 1, 
                function(x) x - newSignal)
  dists = apply(diffs, 2, function(x) sqrt(sum(x^2)) )
  closest = order(dists)
  return(trainSubset[closest, 1:3 ])
}

predXY = function(newSignals, newAngles, trainData, 
                  numAngles = 1, k = 3){
  
  closeXY = list(length = nrow(newSignals))
  
  for (i in 1:nrow(newSignals)) {
    trainSS = selectTrain(newAngles[i], trainData, m = numAngles)
    closeXY[[i]] = 
      findNN(newSignal = as.numeric(newSignals[i, ]), trainSS)
  }

  estXY = lapply(closeXY, 
                 function(x) sapply(x[ , 2:3], 
                                    function(x) mean(x[1:k])))
  estXY = do.call("rbind", estXY)
  return(estXY)
}
